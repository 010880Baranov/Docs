Основи CSS

CSS (Cascading Style Sheets) — це код, який ви використовуєте для стилізації вашої веб-сторінки. Основи CSS допоможуть вам зрозуміти, що вам потрібно для початку роботи.

Так что же таке CSS?

Як і HTML , CSS на самій частині не є мовою програмування. Це не мова зміни - це мова таблиць стилів . Це означає, що він дозволяє застосовувати стилі вибору до елементів у документах HTML. Наприклад, щоб вибрати всі елементи абзацу на HTML-сторінці та змінити текст всередині них із червоним кольором, ви повинні записати цей CSS: 

    p {
    color: red;
    }

Давайте попробуйте: додайте ці три строки CSS у новий файл у вашому текстовому редакторі, а потім введіть файл style.cssу вашій папці styles.

Но нам все рівно потрібно змінити CSS у нашому HTML-документі. У протилежному випадку стиль CSS не пов’язує з цим, оскільки ваш браузер відображає HTML-документ.

Відкрийте ваш файл index.htmlі вставте наступну строку куда-нибудь в шапку, між <head> та <head> тегами: <link href="styles/style.css" rel="stylesheet" type="text/css">

Использование CSS в HTML

Язык стилей CSS не связан с HTML напрямую. Необходимо указать браузеру, откуда нужно загрузить и интерпретировать стили. Для этого есть три способа:

  • Указать стили в качестве значения атрибута style. Такую запись называют «инлайн» (inline), что означает запись правила в одну строку с использованием атрибута style.
  • Использовать специальный тег <style>, который указывают в секции head . Браузер обработает всё, что написано внутри этого тега как CSS код
  • Подключать отдельный файл со стилями. В этом случае создаётся файл с расширением .css, в котором записываются CSS правила. Для подключения файла используется мета-тег <link>

Какой бы способ подключения стилей вы не выбрали, синтаксис правил остаётся неизменным: название-свойства: значение; . Важно не забывать добавлять символ ; после значения свойства. Так браузер сможет отделить правила друг от друга.

CSS: Класи

Основним рішенням для стилізації певної частини сторінки є використання класів. Вони дозволяють нам встановлювати потрібний стиль лише для елементів, які мають задоволений необхідний клас, а не для всіх.

Щоб задати стиль для класу в CSS, ми використовуємо селектор `.name-class' . Зверніть увагу на крапку, саме так браузер розуміє, що цей стиль вказано для класу. 

            <p class="paragraph">
               The text of our paragraph.
               It's the paragraph class in CSS that
                will set properties for this block of text.
              <p>
              .paragraph {
               color: #333333;
               /*
                 The color property sets the color of the text.
                 In this case, it is gray with a value of #333333
                */
                }

Оскільки на сторінці може бути багато класів з однаковою назвою, це правило працюватиме для всіх елементів, де є атрибут класу paragraph

CSS: селектори

Ми вже бачили селектори тегів і класів у навчальних посібниках. Селектор — це опис того, до чого слід застосувати певний стиль CSS. Вони можуть бути досить складними, вказуючи не тільки на конкретний елемент, а й на те, де він розташований або в якому тегу він знаходиться. Пам’ятайте про селектор класів. Це стосується будь-якого елемента, клас якого ми вказали. 

            <p class="text-bold">Bold text inside a paragraph<p>
          
            <div class="text-bold"> Another text inside a simple block element. <div>
            .text-bold {
            font-weight: bold;
            }

Властивість font-weight зі значенням жирного шрифту встановить текст напівжирним. Ви дізнаєтеся більше про цю властивість, коли вивчатимете форматування тексту. Найважливіше зараз те, що цей стиль буде застосовано як до абзацу, так і до елемента блоку <div>, оскільки вони мають однаковий клас.

Ще одним знайомим селектором є селектор тегів. Його мета — встановити стилі для всіх тегів на сторінці. Це дуже корисно, якщо ви хочете встановити початкові стилі для тегів у всьому проекті. Наприклад, щоб встановити шрифт, його розмір і товщину. 

            p {
            color: #333;
            font: 18px/1.5 sans-serif;
          }

Використовуйте селектор тегів лише для встановлення глобальних правил у всьому проекті. Це важливо, оскільки стилі будуть застосовані до всіх тегів одночасно, і ви можете випадково перезаписати наявні стилі.

Інший спосіб вказати стилі — це використовувати Ідентифікатор. Використовуючи id атрибут, ви можете вказати унікальну назву для будь-якого елемента на сторінці. Якщо вказано це ім’я, то на елемент можна посилатися за допомогою ідентифікаційного селектора. #identifier_name Для цього використовується селектор форм 

            
Red paragraph

          #red {
            color: red;
          }

Важливо пам’ятати, що значення ідентифікатора є унікальним. На сторінці може бути лише один елемент з одним конкретним значенням. Наприклад, у наведеному вище прикладі ідентифікатор redуже використовувався в абзаці. На сторінці не може бути другої версії цього елемента. З цієї причини ідентифікатори зазвичай вказуються для великих обгорток, таких як головний заголовок сайту, але це не є обов’язковою умовою

CSS: Пріоритет

Говорячи про каскад, вам може бути цікаво, що станеться, якщо різні джерела мають однакові правила для елемента, але з різними значеннями. У цьому випадку існують правила, які визначають пріоритети стилів з різних джерел. У порядку важливості список виглядає так:

  • Стилі в атрибуті тегу
  • Стилі за замовчуванням, додані браузером
  • Стилі в окремому файлі

Значення, зазначені в styleатрибуті, будуть важливішими за властивості в <style> тегу, і вони будуть важливішими, ніж стилі браузера за замовчуванням.

Пріоритетні селектори

Важливою особливістю CSS є те, що різні типи селекторів мають різні пріоритети. Якщо стилі суперечать один одному в різних селекторах, вступає в дію принцип пріоритету.

Давайте розглянемо приклад. Давайте створимо абзац з класом як red і ідентифікатором як blue.

<p id="blue" class="red">Text with the class red and identifier blue </p> Давайте додамо конфліктні стилі для класу, ідентифікатора та тегу: 

        p {
          color: black;
          }

        .red {
         color: red;
          }

        #blue {
         color: blue;
        }

Якого кольору буде абзац? Спробуйте запустити цей приклад у редакторі. Ви побачите, що текст синій. Яке відношення це має до чогось?

У селекторів також є пріоритети. Зазвичай ви можете розмістити селектори в такому порядку пріоритету:

  • Вибір за ідентифікатором ( #blue)
  • Вибір за класом ( .red)
  • Вибір за тегом ( p)

Де 1 – найвищий пріоритет, а 3 – найнижчий.

З цих правил можна зрозуміти, що властивості, зазначені в селекторе за ідентифікатором, матимуть більший пріоритет, ніж стилі в селекторі за тегом і класом. Тому абзац в останньому прикладі був синім.

CSS: Кольори

Використовуючи CSS, ви можете встановити колір тексту для всієї сторінки або для невеликої області. Це визначається обраним вами селектором. Сам колір встановлюється за допомогою властивості color, значенням якої, природно, є колір.

Увага! Важлива інформація Якщо ви хочете встановити колір тексту для всієї сторінки, ви можете використовувати bodyселектор. Властивість colorуспадковується, тому колір буде встановлено для всіх текстових елементів: 

            body {
            color: #333333;
            }

CSS: вирівнювання тексту

Найефективніший спосіб виділити частину тексту — вирівняти його інакше, ніж решту тексту. Багато сайтів вирівнюють заголовок по центру, щоб користувач міг швидше розрізнити його.

CSS дозволяє вирівнювати текст за допомогою text-align властивості з 4 основними значеннями:

  • left. Цей використовується за замовчуванням
  • center
  • right
  • justify
              <p class="left">Left edge text></p>
              <p class="center">Center text></p>
              <p class="right">Right edge text></p>
              <p class="justify">Justified. It is rarely used, because it leaves unequal spaces between words</p>
            
              .left {
            text-align: left;
              }

            .center {
              text-align: center;
            }

            .right {
              text-align: right;
            }

            .justify {
              text-align: justify;
            }

CSS: Розмір шрифту

Інший спосіб розділити розділи тексту — використовувати різні розміри шрифту. Ми вже бачили, що браузери стилюють заголовки по-різному. Це розмір тексту, який відокремлює h1 заголовок від h2 заголовка тощо.

Ви можете керувати розміром тексту в CSS за допомогою font-size властивості, яка приймає числове значення та одиницю вимірювання. Наразі ми будемо використовувати пікселі як одиницю виміру, хоча є кілька інших важливих одиниць, які ви дізнаєтеся до кінця цього курсу. 

            <p>Warning, this paragraph has the standard font size</p>
            <p class="warning";>This paragraph has a font size of 24 pixels</p>

          .warning {
            font-size: 24px;
            }

CSS: Вага шрифту

У курсі HTML тег b мав ту ж роль, що й font-weight властивість у CSS, а саме, звичайно, керувати вагою шрифту. Властивість є більш гнучким, ніж стандартні стилі b тега, оскільки ви можете зробити його не тільки товщі, але й тонше.

Є два особливо поширені значення для font-weight властивості:

  • bold— товстіший шрифт. його текст буде виглядати так само, як якщо б ви використовували тег strong або b
  • normal є значенням за замовчуванням. Це корисно, якщо весь текст має нестандартну вагу шрифту, але розділ повинен мати стандартну вагу 
            <p class="bold">>Text with bold typeface</p>
            
            .bold {
              font-weight: bold;
            }

CSS: курсивний шрифт

Текст курсивом часто використовується для підкреслення цитат або термінів. На відміну від зміни товщини, курсив дозволяє підкреслити частину тексту, але не виділити його, щоб він відразу став більш помітним.

Щоб змінити стиль шрифту, використовуйте font-style властивість, яка може приймати одне з таких значень:

  • normal- нормальний виведення тексту.
  • italic- курсив
  • oblique- курсив

CSS: Горизонтальні лінії

Підкреслення часто використовується для посилення тексту. Він показує важливість поточного слова або тексту (і також часто використовується для відображення посилань у документі). Іншим способом оформлення тексту є використання закреслення, щоб показати неправильний фрагмент тексту, наприклад, «Не робіть того чи того ».

Властивість text-decorationз трьома основними властивостями використовується для такого типу модифікації тексту:

  • underline- підкреслити текст
  • line-through- Перемалюйте текст
  • overline- надписний текст

CSS: висота рядка

Висота рядка (міжрядковий інтервал) важлива під час роботи з текстом. Цей параметр визначає, скільки місця буде між рядками.

Щоб задати висоту лінії, ми використовуємо line-height властивість. Значення — це число і одиниця виміру. Як і у випадку встановлення розміру шрифту, ви можете використовувати різні одиниці вимірювання.

CSS: Сімейство шрифтів

Багатьом людям знайома фраза «гратися зі шрифтами», це чудово пам’ятати, коли ви не можете змусити текст виглядати правильно. Щоб задати шрифт, використовується font-family правило, що дозволяє вибрати один або кілька шрифтів, які будуть використовуватися в документі. 

            <style>
            .new-font {
              font-family: Arial, Futura;
            }
            </style>
          
            <p class="new-font">Paragraph</p>

Ви можете використовувати шрифти, встановлені у вашій системі. Ви повинні вибрати «загальні шрифти», оскільки інші користувачі можуть не мати інших шрифтів. Також можна включити шрифти зі сторонніх служб, таких як Google Fonts , або завантажити їх безпосередньо на сервер, де розташований сайт.

Під час додавання стилів рекомендується додавати до правила універсальне сімейство шрифтів font-family. На даний момент існує 5 таких сімей:

  • serif- шрифти із засічками. Times New Roman є яскравим прикладом
  • sans-serif- шрифти без засічок. Найвідомішими з цих шрифтів є Arial і Verdana .
  • cursive- курсивні шрифти
  • fantasy- декоративні шрифти. Це сімейство використовується найменш часто. Це тому, що декоративні шрифти занадто різні, щоб бути взаємозамінними
  • monospace- моноширинні шрифти. До них належать шрифти, у яких усі символи мають однакову ширину. Вони часто використовуються програмістами в текстових редакторах

Додавши до правила універсальне сімейство шрифтів font-family, ми можемо як би застрахуватися від того, що користувач не матиме вказаного нами шрифту. Браузер автоматично вибере заміну для відсутнього шрифту із заданого універсального сімейства шрифтів.

CSS: узагальнене правило для шрифтів

Зручний спосіб налаштувати властивості шрифту - це використовувати font правило, яке дозволяє вказати:

  • font-style
  • font-variant
  • font-weight
  • font-size/ line-height(ці два правила записуються косою рискою)
  • font-family

Шість різних правил в одному! Це допомагає скоротити код CSS і покращити його читабельність. Обов'язковими є font-size і font-family. Решта можна пропустити.

Приклад запису з усіма можливими правилами: font: italic small-caps bold 24px/1.2 Arial;

CSS: Межі

Будь-який елемент блоку в HTML можна виділити фреймами. Цей стилістичний засіб дозволяє відокремити монохромні компоненти один від одного або виділити ключовий елемент на сторінці

Цей текст обрамлений, тому його легко знайти на сторінці

Щоб створити межу для елемента, ми використовуємо border властивість, яка є скороченою для кількох властивостей:

  • border-width
  • border-style
  • border-color

Ви можете вказати кілька властивостей або об’єднати все всередині borderвластивості. Це найпоширеніший варіант і виглядає так: 


            .element {
            border: 1px solid #ccc;
          }

          де:
          
          1px- ширина межі
          solid- стиль бордюру
          #ccc- колір межі

Значення пікселів і шістнадцятковий колір, з якими ми вже мали справу в цьому курсі, але в CSS є вісім стилів меж:

  • dotted
  • dashed
  • solid
  • double
  • groove
  • ridge
  • inset
  • outset

а також є none значення, яке «видаляє» межу, тому що за допомогою border-style: none браузери ігнорують інші властивості та видаляють межу

CSS: Фон

Поки ми вивчали CSS, ми вже стикалися з налаштуванням власного кольору для тексту всередині блоку. Але це не єдиний варіант налаштування кольору - ви можете встановити його як фон для всього блоку.

Зверніть увагу, що область з теорією та редактором на уроках відокремлена від решти сайту білим фоном. Це зроблено не тільки для зручності читання, а й для того, щоб допомогти очам швидко знайти область з основним вмістом. Щоб задати колір фону, ми використовуємо властивість background-color , значенням якої, звичайно, є колір.

CSS: padding

Дивлячись на різні блоки, які ми створили під час наших уроків, ви помітите, що текст «прилипає» до країв блоків.

Для створення внутрішніх відступів використовуються чотири правила:

  • padding-top
  • padding-right
  • padding-bottom
  • padding-left

Якщо ви хочете встановити відступи з усіх боків, не використовуйте 4 різні властивості, вам потрібен лише один - padding .

На додаток до цього позначення існують скорочені позначення цих правил для зручності:

  • Якщо ви вкажете лише одне значення, воно буде використовуватися одночасно для всіх сторін
  • Якщо ви вкажете два значення, перше використовуватиметься для вертикального (верхнього і нижнього) полів, а друге — для горизонтального (правого і лівого) полів.
  • Якщо вказати три значення, вони використовуватимуться для верхнього, горизонтального та нижнього полів

CSS: margin

Внутрішні відступи дозволили вам додати відступ до блоку та пробіл між межею блоку та його вмістом. Але як утримати два сусідні блоки від тиснення один до одного?

Використовуйте зовнішні відступи. Їх принцип такий же, як і поля, тільки напрямок інший. З точки зору властивостей і їх цінності, це те саме. Є 4 правила і одне загальне:

  • margin-top
  • margin-right
  • margin-bottom
  • margin-left

Щоб об’єднати значення, використовуйте margin властивість, яка зберігає порядок, у якому воно зазначено.

CSS: висота та ширина блоку

Коли ви створюєте сторінки, вам часто потрібно керувати висотою і шириною блоків. Це корисно в довгих статтях, коли ви хочете помістити пояснювальний текст у невеликий блок. Таким чином він буде відокремлений від основної частини тексту, і ви зможете його відразу побачити.

Правила widthі height використовуються для керування висотою та шириною елементів блоку, приймаючи значення для ширини та висоти відповідно (наприклад, у пікселях або інших доступних одиницях вимірювання).

Давайте створимо блок висотою 100 пікселів і шириною 100 пікселів. Щоб візуально відокремити його від теорії в підручнику, встановіть колір фону на чорний: 


            <style>
            .square {
              width: 100px;
              height: 100px;
              background-color: #000;
            }
            </style>
          
            <div class="square"></div>

CSS: Box Model

Використовуючи деякі властивості, такі як border і padding, ви можете помітити, що це впливає на кінцевий розмір блоку. І справді так.

Тобто кінцевий розмір встановлюється не тільки властивостями height, widthа й багатьма іншими:

  • padding- внутрішні поля
  • border- кордони
  • margin- зовнішні відступи

По суті, щоб визначити остаточний розмір елемента, необхідно скласти значення всіх цих властивостей. Наприклад, якою буде ширина цього блоку? 

            .box {
            width: 100px;
            height: 100px;
          
            padding: 20px;
            margin: 10px;
          
            border: 1px solid #fff;
            }

100 пікселів? 120 пікселів? 121 пікселів? Як дізнатися це значення? Ви повинні скласти всі значення, які утворюють блок по горизонталі: 

          width
          padding-left/padding-right
          border-left/border-right
          margin-left/margin-right

Отже, кінцева ширина елемента на сторінці 100px + 20px + 20px + 1px + 1px + 10px + 10px дорівнює 162px.

Така поведінка називається коробковою моделлю . Він описує зв’язок між властивостями та кінцевим розміром елемента. Відповідно до цієї моделі браузери «малюють» блок, накладаючи правила один на одного, перш ніж відобразити елемент на сторінці.

Цю модель варто пам’ятати, тому що при роботі з CSS важливо розуміти, як і де браузер «малює» блок, і як це вплине на його висоту і ширину.

Але ви можете погодитися, що коли ви встановлюєте властивості widthі , ви очікуєте саме цих значень. height Чи можна змінити поведінку коробкової моделі? Так! Для цього скористайтеся властивістю box-sizing , яка приймає такі значення:

  • content-box- значення за замовчуванням. Працює, як описано вище
  • border-box- значення межі та полів не збільшують елемент, а навпаки, «з’їдають» простір у вмісті, тобто властивості не впливатимуть на висоту та ширину. Якщо встановлено фіксоване значення, воно залишиться таким, але всередині буде менше місця для вмісту

CSS: медіа-запити

Для роботи в Інтернеті використовується багато пристроїв з різними розмірами екрана та роздільною здатністю, наприклад комп’ютери, смартфони, планшети, ноутбуки тощо. Ця кількість пристроїв вимагає від розробників можливості налаштувати свої сторінки для різних варіантів роздільної здатності.

Один із способів змінити стилі залежно від роздільної здатності – це використовувати спеціальні правила CSS, які працюють за певних умов.

Правило ЗМІ

Усі медіа-запити мають схожий шаблон і використовують @media конструкцію: 

      @media (Condition) {
        /* Rules */
      }

Які правила наведено всередині @media конструкції? Це може бути що завгодно! Думайте про це як про новий файл CSS з власними селекторами та властивостями. Наприклад, ви можете змінити колір тексту в елементах із класом .text на чорний. Тоді дизайн набуде такого зображення: 


    @media (Condition) {
    .text {
      color: #000000;
    }
  }

Які умови? Припустимо, ви хочете змінити колір тексту, якщо ширина viewport менша або дорівнює 700 пікселям. max-width: 700px Потім потрібно вказати правило . TЦе дуже схоже на звичайне правило CSS, яке визначає, що властивості повинні працювати, якщо viewportширина становить 700 пікселів або менше: 


    @media (max-width: 700px) {
      .text {
       color: #000000;
      }
    }

З появою смартфонів необхідність працювати з тим, як користувач тримає телефон, як вертикально, так і горизонтально, стає дуже актуальною. Коли користувач тримає телефон вертикально, висота екрана перевищує його ширину. При горизонтальному використанні все навпаки – ширина більше висоти. Це вимагає правильного вирівнювання блоків на сторінці, щоб нею було зручно користуватися з будь-якого положення.

Медіа-запити мають спеціальну умову orientation, яка може мати одне з двох значень:

  • portrait- портретна орієнтація пристрою. Висота екрана перевищує його ширину
  • landscape- альбомна орієнтація пристрою. Висота екрана менше його ширини

Нехай текст буде білим для книжкової орієнтації і чорним для альбомної орієнтації: 


      @media (orientation: portrait) {
      .text {
      color: #ffffff;
      }
    }
  
      @media (orientation: landscape) {
      .text {
      color: #000000;
      }
    }

Медіа-запити можуть бути набагато складнішими, з багатьма різними варіаціями.

CSS: анімація

Анімація — це великий модуль CSS. На жаль, неможливо описати всі особливості за один урок. У цьому уроці ми покажемо вам основний принцип анімації в CSS на основі @keyframes. Наприклад, давайте зробимо текстову анімацію, де колір тексту змінюється з синього на червоний.

Правило @keyframesта назва анімації будуть використовуватися для встановлення анімації: 


            @keyframes color-change {
            // Inside will be our animation
          }

Назва анімації може бути будь-якою. Ви можете вибрати його самостійно. Після цього animation властивість дізнається, яку анімацію ви хочете відтворити за її назвою.

Всередині @keyframes потрібно вказати різні часові інтервали, в які будуть відбуватися зміни. Наприклад, на початку (0% завершення анімації) має бути синій текст, у середині анімації (50%) він повинен стати червоним, а в кінці (100%) він повернеться до синього . 

            @keyframes color-change {
            0% {
              color: blue;
            }
          
            50% {
              color: red;
            }
          
            100% {
              color: blue;
            }
          }

Тепер нам потрібно застосувати створену нами анімацію до тексту на сторінці. Щоб досягти цього, використовуйте animation правило з трьома основними значеннями:

Назва анімації. Тривалість анімації. Подається в секундах (1с, 2с, 3с тощо) Вам також потрібно вирішити, чи буде анімація нескінченною (якщо вона нескінченна, то потрібно вказати параметр нескінченності) 

            .animation-text {
            animation: color-change 3s infinite;
          }

Залишилося лише створити абзац з необхідним класом, і анімація почне працювати.

CSS: змінні

Змінна створюється за допомогою ---variable-name конструкції. Ім'я змінної можна вибрати самостійно. Як приклад, давайте створимо змінну з назвою --main-color, яка містить базовий колір сторінок. Встановіть чорний колір: 


            --main-color: #000000;

Ця простота приховує той факт, що змінні мають різні області дії, області файлу, з яких їх можна використовувати. Це велика тема, тому давайте поки поговоримо лише про глобальний масштаб. Це дозволить використовувати змінну в будь-якій області файлу CSS.

Щоб створити глобальну змінну, її потрібно вказати в спеціальній конструкції, :root. Зазвичай це робиться на початку файлу CSS: 


            :root {
            --main-color: #000000;
            }

Тепер ви можете використовувати змінну в будь-якій частині нашого коду CSS. Це робиться за допомогою спеціальної конструкції, var(--variable-name). 


            :root {
            --main-color: #000000;
            }

            .news-block {
              background-color: var(--main-color);
            }

            .left-sidebar {
            background-color: var(--main-color);
            }

Якщо ми хочемо встановити інший відтінок чорного, просто змініть значення -main-color змінної, і всі зміни автоматично застосовуватимуться до блоків із класами .news-blockі.left-sidebar

Позиціонування елементів

Позиціонуванням називається положення елемента в системі координат. Розрізняють кілька типів позиционування: нормальное, відносное, абсолютное, фіксоване та приклеєне. В залежності від типу, який встановлюється через свойство position , змінюється я і система координат. Завдяки комбінації властивостей положення position, left , top, right и bottom можна відображати елемент у точці із заданими координатами, фіксувати його в заданому місці, визначати положення одного елемента щодо іншого, керувати розмірами тощо. Подібним іншим властивостям CSS управління позиціонуванням доступно через JavaScript . Таким чином, допустимо динамічно змінювати положення елементів без перезавантаження сторінок, створення анімації та різних ефектів. Свойство position не тільки встановлює тип позиционування, але і змінює поведінку елемента, завдяки чому розширюється арсенал приємів верстки.

  • left — задаёт координаты левого края элемента от левого края родителя или левого края исходного положения элемента.
  • right — задаёт координаты правого края элемента от правого края родителя или правого края исходного положения элемента.
  • top — задаёт координаты верхнего края элемента от верхнего края родителя или верхнего края исходного положения элемента.
  • bottom — задаёт координаты нижнего края элемента от нижнего края родителя или нижнего края исходного положения элемента. Заметьте, что напрямую нельзя задать положение правого края элемента от левого края родителя и наоборот. Также нельзя задать положение верхнего края элемента от нижнего края родителя и наоборот.

У position есть пять значений:

  1. static — нормальное или статичное позиционирование, при этом элементы отображаются на веб-странице в том порядке, в каком они идут в исходном коде HTML сверху вниз. static — это значение по умолчанию для position.
  2. relative — относительное позиционирование. Изменяет положение элемента от его исходного расположения.
  3. absolute — абсолютное позиционирование. Элемент при этом не существует в потоке документа и его положение задаётся относительно краёв браузера.
  4. fixed — фиксированное позиционирование. По своему действию похоже на абсолютное позиционирование, но в отличие от него привязывается к указанной свойствами left , top , right и bottom точке на экране и не меняет своего положения при прокрутке веб-страницы.
  5. sticky — липкое позиционирование. Обычно применяется для фиксации заголовка на одном месте, пока содержимое, к которому относится заголовок, прокручивается на странице.

Свойство position не наследуется, так что для дочерних элементов его требуется указывать явно. Для изменения положения элементов по оси Z применяется свойство z-index , которое непосредственно связано со свойством position. z-index работает только для элементов, у которых position задано как relative , absolute или fixed .

Нормальное позиционирование

Если для элемента свойство position не задано или его значение явно указано как static, элемент выводится в потоке документа как обычно. Иными словами, элементы отображаются на странице в том порядке, как они идут в исходном коде HTML. static — это значение свойства position по умолчанию, иными словами, если вы не задали для элемента position , то оно считается браузером как static .

Формально, static не является типом позиционирования, это естественное исходное состояние любого элемента. Таким образом, термины «нормальное позиционирование», «обычное позиционирование», «статичное позиционирование», «статичный элемент» указывают лишь на значение static у свойства position , поскольку позиционирования как такового нет и элемент остаётся на своём начальном месте.

Для нормального позиционирования характерны следующие особенности:

  • элементы на веб-странице выводятся в том порядке, как они описаны в коде HTML;
  • свойства left , right , top и bottom не дают никакого эффекта;
  • свойство z-index не работает.

Само по себе значение static применяется редко, поскольку является значением по умолчанию и не требует явного указания. Оно используется на практике лишь для возврата свойства position к начальному значению. В примере 2 для элемента задано относительное позиционирование и оно сдвинуто вниз от своего исходного положения. Псевдокласс :hover отменяет относительное позиционирование и элемент возвращается в исходное положение, если навести на него курсор.

Относительное позиционирование

Если задать значение relative для свойства position , то положение элемента устанавливается относительно его исходного места. Добавление свойств left , top , right и bottom изменяет позицию элемента и сдвигает его в ту или иную сторону от первоначального расположения. Положительное значение left определяет сдвиг вправо от левой границы элемента, отрицательное — сдвиг влево. Положительное значение top задаёт сдвиг элемента вниз , отрицательное — сдвиг вверх. Свойства bottom и right производят обратный эффект. При положительном значении right сдвигает элемент влево от его правого края, при отрицательном — сдвигает вправо . При положительном значении bottom элемент поднимается вверх, при отрицательном опускается вниз.

Для относительного позиционирования характерны следующие особенности:

  • элемент можно смещать относительно исходного положения с помощью свойств left , top , right и bottom;
  • при смещении элемента относительно исходного положения, место, которое занимал элемент, остаётся пустым и не заполняется ниже или вышележащими элементами;
  • если сдвинуть элемент вправо за пределы окна браузера, то появится горизонтальная полоса прокрутки;
  • если сдвинуть элемент вниз за пределы окна браузера, то появится вертикальная полоса прокрутки;
  • смещение элемента влево и вверх за пределы окна браузера не оказывает влияния на полосы прокрутки;
  • работает свойство z-index;

этот тип позиционирования неприменим к элементам таблицы вроде ячеек, строк, колонок и др.

Относительное позиционирование часто применяется для создания анимации и эффектов, связанных со сдвигом элементов. C помощью элемента <button> добавляется кнопка с тенью, при наведении курсора на кнопку она смещается вправо и вниз на величину тени, а сама тень при этом прячется. Всё это в комплексе придаёт кнопке трёхмерный эффект.

Для относительно позиционированных элементов работает свойство z-index , которое управляет положением элементов по оси Z. Если требуется задать наложение элементов относительно друг друга, то в стилях достаточно указать position со значением relative без смещения самого элемента.

Абсолютное позиционирование

При абсолютном позиционировании элемент не существует в потоке документа и его положение задаётся относительно краёв браузера. Установить этот тип можно через значение absolute свойства position. Координаты указываются относительно краёв окна браузера, называемого «видимой областью» или «областью просмотра»

Для элемента с абсолютным позиционированием характерны следующие особенности:

  • Ширина элемента, если она не задана явно, определяется шириной содержимого плюс значения padding, border и margin.
  • Элемент не меняет своё исходное положение, если у него нет свойств right, left, top и bottom.

Одновременно указанные свойства left и right формируют ширину элемента, но только если width не указано. Стоит добавить свойство width и значение right будет проигнорировано. Аналогично произойдёт и с высотой элемента, но уже участвуют свойства top , bottom и height.Свойства left и top имеют более высокий приоритет по сравнению с right и bottom . Если left и right противоречат друг другу, то значение right игнорируется. То же самое касается и bottom . Если left задать отрицательное значение, то элемент уйдёт за левый край браузера, полосы прокрутки при этом не возникнет. То же относится и к свойству top , только элемент уйдёт за верхний край. Если left задать значение больше ширины видимой области или указать right с отрицательным значением, то появится горизонтальная полоса прокрутки. Подобное правило работает и с top, только речь пойдёт о вертикальной полосе прокрутки. Элемент перемещается вместе с документом при его прокрутке. Работает свойство z-index , при этом абсолютно позиционированные элементы перекрывают статичные элементы, даже если они в коде ниже. Свойство position со значением absolute можно использовать для создания эффекта фреймов. Кроме абсолютного позиционирования для элементов следует назначить свойство overflow со значением auto . Тогда, если содержимое превысит высоту видимой области, появится полоса прокрутки. Высота и ширина «фреймов» формируется автоматически путём одновременного использования свойств left , right для ширины и top , bottom для высоты.

Тот же самый приём, когда мы задаём размеры элемента с помощью свойств left , right,top , bottom применяется для создания разных эффектов. К примеру, можно сделать затемнение веб-страницы для вывода сообщения и привлечения к нему внимания. В примере 2 добавляется элемент с классом overlay , размеры которого занимают всю область просмотра. Делаем его полупрозрачного чёрного цвета, а поверх отображаем блок с классом dialog , который выводится по центру горизонтали.

Обратите внимание, что для вывода абсолютно позиционированного элемента по центру горизонтали применяется следующий стиль: 

.dialog {
  width: 260px;
  left: 0; right: 0;
  margin-left: auto; margin-right: auto;
  }

Таким образом, сочетание width, left, right и margin управляет выравниванием элементов.

Абсолютное позиционирование позволяет также скрывать элементы. Для этого часто добавляется стиль left: -9999px, который переносит элемент далеко влево за область просмотра. В частности, этот приём используется для стилизации переключателей и флажков, когда мы скрываем настоящий флажок, а вместо него стилизуем элемент <label>, связанный с <input> и выполняющим его функции.

relative и absolute

Обычно относительное позиционирование само по себе применяется не часто, поскольку для смещения элемента от его исходного положения есть ряд других свойств, к примеру, margin или transform. Но комбинация разных типов позиционирования для вложенных элементов является одним из удобных и практичных приёмов вёрстки. Если для родительского элемента задать relative, а для дочернего absolute, то произойдёт смена системы координат и положение дочернего элемента при этом указывается относительно его родителя.

Фиксированное позиционирование

Фиксированное позиционирование элемента задаётся значением fixed свойства position и по своему действию похоже на абсолютное позиционирование. Но в отличие от него привязывается к указанной свойствами left, top, right и bottom точке на экране и не меняет своего положения при прокрутке веб-страницы.

Для фиксированного позиционирования характерны следующие особенности.

  • При прокрутке веб-страницы элемент остаётся на своём исходном месте.
  • Положение элемента всегда отсчитывается от области просмотра браузера, независимо от позиционирования родителя.
  • При выходе элемента за пределы видимой области справа или снизу от неё, не возникает полос прокрутки и элемент просто «обрезается».

В остальном, значение fixed похоже на absolute .

Применяется такой тип позиционирования для создания меню, вкладок, заголовков, в общем, любых элементов, которые должны быть закреплены на веб-странице и всегда видны посетителю.

Свойства блока

Любой элемент на веб-странице представляет собой прямоугольный блок, который состоит из набора свойств, подобно капустным листам накладываемых друг на друга. Основой блока выступает его содержимое (это может быть текст, изображение, видео и др.), ширина которого задается свойством width, а высота через height; вокруг содержимого идут поля (padding), они создают пустое пространство от содержимого до внутреннего края границ; затем идут границы (border), а вокруг границ отступы (margin) — невидимое пустое пространство от внешнего края границ. Порядок этих свойств в блоке чётко определён и не может быть нарушен.

Свойство padding

Основное предназначение padding — создать пустое пространство вокруг содержимого элемента, например текста, чтобы он не прилегал плотно к краю элемента. Использование padding повышает читабельность текста и улучшает внешний вид страницы.

Свойство border

Свойство border добавляет вокруг элемента рамку заданной толщины, стиля и цвета. Для создания линий на отдельных сторонах элемента используются свойства border-top, border-right, border-bottom и border-left, соответственно устанавливающие линию сверху, справа, снизу и слева. Так же эти свойства могут быть использованы и чтобы убрать линию на определённой стороне. 

border: 2px solid #0069b5; /_ Параметры рамки _/
              border-left: none; /_ Убираем линию слева _/
              border-right: none; /_ Убираем линию справа _/

Здесь имеет значение порядок свойств — сперва устанавливаем рамку, и лишь после этого убираем линии на нужных сторонах.

Сама рамка идёт после содержимого элемента и пустого пространства, создаваемого padding . Если свойство padding не задано, то рамка начинается сразу вокруг содержимого.

Свойство border является частью блока и влияет на его размеры. Альтернативой border выступает свойство outline. Вот его особенности:

  • outline не влияет на размеры и положение самого элемента;
  • outline не занимает место, не влияет на окружающие элементы и отображается поверх них;
  • для outline нельзя задать параметры линии на отдельных сторонах элемента, outline применяется сразу ко всем четырём сторонам;
  • свойство border-radius, задающее радиус скругления рамки, не действует на outline;
  • outline можно расширять и сжимать с помощью outline-offset, отрицательное значение выводит рамку внутри элемента.

Свойство outline не заменяет border и вполне может существовать вместе с ним.

Свойство margin

Свойство margin устанавливает пустое пространство от внешнего края border, padding или содержимого блока. Под margin нет своего фона и он принимает фон родительского элемента.

margin в основном используется для создания вертикальных и горизонтальных отступов между элементами. Аналогично другим блочным свойствам есть свойства для каждой стороны — margin-top, margin-right, margin-bottom и margin-left, соответственно устанавливающие отступ сверху, справа, снизу и слева.

Значения margin. В качестве значений margin используются любые допустимые единицы длины, к примеру, пиксели, проценты, em, rem и др. В отличие от padding свойство margin может быть отрицательным (например: -10px), а также принимать значение auto. В остальном margin похоже на padding, у него также может быть от одного до четырёх значений.

Одно значение — определяет отступы для всех сторон блока.

  margin: все стороны;
  margin: 10px;

Два значения — первое определяет отступы сверху и снизу для элемента, второе слева и справа для элемента. 

  margin: верх-низ лево-право;
  margin: 10px 20px;

Три значения — первое задаёт отступ сверху для элемента, второе одновременно слева и справа, а третье снизу. 

    margin: верх лево-право низ;
    margin: 10px 20px 5px;

Четыре значения — первое определяет отступ сверху, второе справа, третье снизу, четвёртое слева. Для запоминания последовательности можно представить часы — значения идут по часовой стрелке, начиная с 12 часов. 

  margin: верх право низ лево;
  margin: 5px 10px 15px 20px;

Значение auto

Свойство margin позволяет выравнивать элемент по центру горизонтали, если использовать значение auto и задать ширину элемента через width. Аналогично можно выровнять по центру горизонтали изображение, для этого даже не надо указывать ширину картинки, поскольку браузер получает её автоматически. Следует только превратить изображение в блочный элемент через свойство display.

Проценты

В качестве значения margin можно использовать процентную запись, с которой связаны следующие особенности.

  • По горизонтали проценты считаются от ширины всего блока.
  • По вертикали проценты считаются от ширины всего блока.

Таким образом, margin в процентах берётся от ширины всего блока, даже для margin-top и margin-botom.

Отрицательный margin

margin может быть задан с отрицательным значением, тем самым элемент сдвигается в противоположном направлении. К примеру, margin-top:-10px поднимает блок вверх на 10 пикселей, а margin-left:-10px сдвигает блок влево.

Обнуление margin

Для некоторых элементов браузер добавляет margin автоматически. Вот несколько примеров: 

          для <body> по умолчанию margin задан как 8px;
          для списков <ul>, <ol>, <dl> margin-top и margin-bottom заданы как 1em;
          для <blockquote> и <figure> margin-left и margin-right заданы как 40px.

Эти margin, согласно макету, не всегда нужны, поэтому их можно обнулить, иными словами, задать для margin нулевое значение. Показано обнуление margin для элемента <body>, после чего элемент <header> плотно прилегает к краям браузера.

CSS: Горизонтальні лінії

Підкреслення часто використовується для посилення тексту. Він показує важливість поточного слова або тексту (і також часто використовується для відображення посилань у документі). Іншим способом оформлення тексту є використання закреслення, щоб показати неправильний фрагмент тексту, наприклад, «Не робіть того чи того ».

Властивість text-decoration з трьома основними властивостями використовується для такого типу модифікації тексту:

  • underline — підкреслити текст
  • line-through — Перемалюйте текст
  • overline — надписний текст

Значення none використовується для видалення символів підкреслення з посилань. Це часто використовується для видалення символу підкреслення з посилань, які є там за замовчуванням.
