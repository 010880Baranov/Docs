Hello GIT!

Команды в Git Bash консоли

clear — Очистить консоль

Навигация

pwd — Показать текущий каталог

ls - Показать файлы в данной папке, кроме скрытых

ls -f — Показать файлы в данной папке, включая и скрытые

cd c:/ — Перейти в конкретный каталог

cd - — Вернуться назад

cd .. — Выйти на 1 уровень вверх

cd ../.. — Выйти на 2 уровня вверх

Создание каталогов

mkdir — Создать папку

cd !$ — Перейти в только что созданную папку

mkdir -p {app1,app2} — Создать сразу несколько папок

mkdir -p app/{css,js} — Создать сразу несколько вложенных папок

Создание файлов

touch index.html — Создать файл index.html

touch app/{css/main.css,js/main.js,index.html} — Создать сразу несколько файлов, никаких лишних пробелов быть не должно

Удаление файлов touch — позволяет создавать файлы

rm test — Удалить пустую папку test

rm -r test — Удалить папку test с файлами внутри неё

Перемещение файлов

mv app1/*.* app2 — Переместить все файлы из папки app1 в папку app2

Команди Git

git config
    
Використання: git config –global user.name “[name]”  

Використання: git config –global user.email “[email address]”  

Ця команда встановлює ім’я автора та адресу електронної пошти відповідно для використання з вашими коммітами.

git init

Використання: git init [repository name]
Ця команда використовується для запуску нового сховища.

git clone 
Використання: git clone [url]  
Ця команда використовується для отримання сховища з існуючої URL-адреси.

git add 
Використання: git add [file]  
Ця команда додає файл до проміжної області.

git add -n \_ Проверка, какие файлы будут добавлены в репозиторий.

git add readme.txt Добавить в репозиторий файл readme.txt(1).

git add util/\*\*/_.cpp Добавить все файлы _.cpp, которые находятся в папке util и её подкаталогах.

git add _.c _.h _.s _.asm Добавить в репозиторий все файлы(1) с расширениями _.c, _.h, \_.s, \*.asm.
    
git commit
Використання: git commit -m “[ Type in the commit message]”  
Ця команда постійно записує або знімає файл в історії версій.

Використання: git commit -a  
Ця команда фіксує будь-які файли, які ви додали за допомогою команди git add, а також усі файли, які ви змінили з тих пір.
    
git diff
Використання: git diff
Ця команда показує відмінності файлів, які ще не поставлені.

Використання: git diff –staged 
Ця команда показує відмінності між файлами в проміжній області та останньою наявною версією.

Використання: git diff [first branch] [second branch]  
Ця команда показує відмінності між двома згаданими гілками.

git reset
Використання: git reset [file]  
Ця команда знімає файл із стадії, але зберігає вміст файлу.

Використання: git reset [commit]  
Ця команда скасовує всі коміти після зазначеного коміту та зберігає зміни локально.

Використання:  git reset –hard [commit] Ця команда відкидає всю історію та повертається до вказаного коміту.

git status
Використання: git status  
Ця команда перераховує всі файли, які потрібно зафіксувати.

git status --untracked-files=no 
Показать состояние репозитория без отображения файлов, которые еще не добавлены в репозиторий.

git rm
Використання: git rm [file]  
Ця команда видаляє файл із вашого робочого каталогу та виконує видалення.

git rm --cached имя файла 
Убирает из репозитория указанный файл (он больше не отслеживается). В рабочем каталоге этот файл останется.

git log
Використання: git log  
Ця команда використовується для відображення історії версій для поточної гілки.

Використання: git log –follow[file]  
Ця команда відображає історію версій файлу, включаючи перейменування файлів.

git show
Використання: git show [commit]  
Ця команда показує метадані та зміни вмісту вказаного коміту.

git tag
Використання: git tag [commitID]  
Ця команда використовується для надання тегів зазначеному коміту.

git branch
Використання:  git branch 
Ця команда містить список усіх локальних гілок у поточному сховищі.

Використання: git branch [branch name]  
Ця команда створює нову гілку.

Використання: git branch -d [branch name]  
Ця команда видаляє гілку функції.

git branch -m old new 
Переименовать ветку. Ветка с именем old получит имя new

git checkout
Використання: git checkout [branch name]  
Ця команда використовується для перемикання з однієї гілки на іншу.

Використання: git checkout -b [branch name]  
Ця команда створює нову гілку, а також перемикається на неї.

git merge
Використання: git merge [branch name]  
Ця команда об’єднує історію вказаної гілки з поточною гілкою.
git remote
Використання: git remote add [variable name] [Remote Server Link]  
Ця команда використовується для підключення вашого локального сховища до віддаленого сервера.

git push
Використання: git push [variable name] master  
Ця команда надсилає внесені зміни головної гілки до вашого віддаленого сховища.

Використання: git push [variable name] [branch]  
Ця команда надсилає коміти гілки до вашого віддаленого сховища.

Використання: git push –all [variable name]  
Ця команда переміщує всі гілки до вашого віддаленого сховища.

Використання: git push [variable name] :[branch name]  
Ця команда видаляє гілку у вашому віддаленому сховищі.

git pull
Використання: git pull [Repository Link]  
Ця команда отримує та об’єднує зміни на віддаленому сервері у ваш робочий каталог.

git stash
Використання: git stash save  
Ця команда тимчасово зберігає всі змінені відстежувані файли.

Використання: git stash pop  
Ця команда відновлює останні заховані файли.

Використання: git stash list  
Ця команда містить список усіх прихованих наборів змін.

Використання: git stash drop  
Ця команда відкидає останній прихований набір змін.

switch  - Отмена предыдущей операции переключения между ветками.

git config --global alias.st "status -s"
После этого команды git st и git status будут выполняться одинаково.
